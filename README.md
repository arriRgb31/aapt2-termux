# AAPT2-Termux

![Platform](https://img.shields.io/badge/Platform-Termux-green)
![Architecture](https://img.shields.io/badge/Architecture-ARM64-blue)
![Android](https://img.shields.io/badge/Android-11%2B-success)
![License](https://img.shields.io/badge/License-MIT-yellow)

Native ARM64 build of **Android Asset Packaging Tool 2 (AAPT2)** for **Termux**.

This project adapts the Android Open Source Project (AOSP) implementation of AAPT2 into a self-contained repository that can be built directly on Android using the native Termux environment.

No separate AOSP checkout is required.

---

# Features

- Native ARM64 executable
- Built entirely inside Termux
- Self-contained repository
- Latest AOSP AAPT2 source
- Clang/LLVM toolchain
- CMake + Ninja build system
- Automatic generation of required build version sources
- Automatic protobuf code generation
- Android 11+ compatible

Make executable:



# Build Pipeline

The repository builds the final executable using the following pipeline.

```
build.sh
    │
    ├── generates
    │      generated/build/version.h
    │      generated/build/version.cpp
    │
    ├── configures CMake
    │
    ├── generates Ninja build graph
    │
    ├── Ninja compiles repository sources
    │
    ├── archives static libraries
    │
    └── links

            build/aapt2
```

The build script performs four primary stages:

1. Generate repository-local build version sources.
2. Configure the CMake project.
3. Generate Ninja build rules.
4. Build the final executable.

---

# Executable Structure

The final executable is declared by CMake as:

```
add_executable(aapt2
    third_party/aapt2/Main.cpp
)
```

The executable links:

```
aapt2
    └── aapt2lib
```

All remaining repository libraries are introduced through `aapt2lib`.

# Generated Sources

The repository generates two categories of source files during the build.

## Build Version Sources

Generated by:

```
build.sh
```

Outputs:

```
generated/build/version.h
generated/build/version.cpp
```

These files are compiled into the `androidfw` static library.

---

## Protobuf Sources

Generated by:

```
protobuf_generate_cpp(...)
```

Input:

```
third_party/aapt2/*.proto
```

Generated outputs:

```
*.pb.cc
*.pb.h
```

The generated protobuf source files are compiled into `aapt2lib`.

# Repository Evidence Boundary

This repository documents the build configuration and dependency graph.

Repository files can prove:

- build graph
- compile graph
- archive graph
- declared link graph
- generated source graph
- source ownership
- dependency declarations

Repository files alone cannot prove:

- final ELF symbol table
- extracted archive members
- linker resolution
- relocation results
- stripped symbols
- runtime dynamic dependencies

Those require inspection of a built executable using tools such as:

- `readelf`
- `llvm-nm`
- `nm`
- `objdump`
- linker map files
